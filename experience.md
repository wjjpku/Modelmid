# 数学作业判别模型探索与经验总结 (Experience)

## 1. 目标与背景
本项目旨在区分和判别各种来源的数学解答：**真实人类手写 (Human) vs 大语言模型 (Deepseek, Kimi)**。这在目前的教育环境中，对于防止滥用生成式 AI 具有重要意义。

## 2. 探索与迭代过程

### 阶段一：数据清洗与标准化设计
- **挑战**：原始数据为含有各种 `\documentclass` 和自定义宏的 LaTeX 源码，且不同学科的排版并不统一。最初引入了《常微分方程》数据，但发现其缺少对应的人类解答。
- **解决**：为了确保有监督学习的基础数据质量，果断去除了常微分方程，仅保留《近世代数》部分，使得每道题目都能提取出标准的人类解答。
- **规范建立**：定义了严格的 CSV 标准 (`id`, `course`, `content`, `human`, `deepseek`, `kimi`)。

### 阶段二：自动化生成大模型解答
- **挑战**：单线程通过 API 请求大模型获取 170+ 道数学推导题非常耗时。
- **解决**：利用 Python 的 `concurrent.futures.ThreadPoolExecutor` 编写了并发请求脚本 (`generate_deepseek_answers.py` 和 `generate_kimi_answers.py`)，并设计了鲁棒的 Prompt（要求扮演数学教授、纯 LaTeX 输出、不带寒暄）。这使得批量数据集构建速度提升了数倍。

### 阶段三：发现并修复数据泄露 (Data Leakage) Bug
- **挑战**：在第一版二分类（Human vs Deepseek）模型中，准确率达到了惊人的 99.7%。但在特征重要性分析中发现，`num_textbf` 特征占据了绝对的主导地位（重要性 > 30%）。经排查，原作者在写解答时习惯用 `\textbf{证}` 起手，而大模型不会这么做。这会导致模型“走捷径”，泛化能力极差。
- **解决**：在 `process_data.py` 中增加了正则清洗规则，去除了诸如 `\textbf{证}` 等个人习惯前缀。修复后，模型的准确率仍然保持在 99%，但特征重要性分布变得非常健康。

### 阶段四：特征工程与模型选择
- **特征设计**：由于我们判别的是排版与逻辑习惯，而不是具体的数学知识，我们没有盲目使用深度学习模型，而是提取了以下领域特异性特征：
  1. 换行与长度特征（行数、平均行长、总字数）。
  2. 数学公式分布（公式块数量、公式密度）。
  3. 逻辑词频密度（因为、所以、显然、不妨设等）。
- **模型评估**：
  - 传统 TF-IDF 效果有限（85% 左右）。
  - 加入自定义排版和逻辑特征后，随机森林与 SVM 都能达到 98%~99% 的准确率。
  - **最终选择**：**组合特征 (TF-IDF + 自定义特征) + SVM 分类器**，因为其在三分类下表现最稳定。

### 阶段五：三分类 (Human vs Deepseek vs Kimi)
- **挑战**：区分人与机器相对容易，但区分不同的大模型（Deepseek 和 Kimi）是否有足够的信息量？
- **发现**：加入 Kimi 数据后，模型整体准确率稳定在 98%。通过随机森林特征分析，我们发现除了“换行习惯”外，**“逻辑词密度”** 成为了区分 Deepseek 和 Kimi 的重要签名。不同的大模型在展开数学推理时，使用连词的偏好截然不同。

## 3. 经验与后续建议
1. **不要盲信高准确率**：在 NLP 文本分类中，一定要去 review 特征重要性，警惕类似 `\textbf{证}` 这样的数据泄露。
2. **结构特征优于内容特征**：判别 LLM 的关键往往不在于它写了什么字，而在于它的“换行、公式包裹、长度、逻辑连词”等结构性习惯。
3. **扩展性**：当前的脚本结构极具扩展性，未来可以轻易加入诸如 ChatGPT、文心一言等新的列进行多分类对抗训练。
