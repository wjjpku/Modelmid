# 实验经验记录 (Experience Log)

## 动态迭代对抗实验 (Iterative Adversarial Experiment) - 2026-04-17

**背景**：
在静态防检测实验（Zero-shot Anti-Detection）取得了一定成功后，为了进一步测试高精度特征分类器的鲁棒性，引入了“基于反馈的动态迭代对抗”机制（LLM-as-an-Optimizer）。我们以在静态防检测中最难被攻破的 **Kimi (Moonshot)** 作为生成器，并由 **DeepSeek** 作为优化器来进行深度测试。

**设计思路 (两个阶段的对比实验)**：

### 阶段一：先验主导型 Optimizer (Prior-Driven Optimizer)
1. **策略**：最初的 Optimizer 提示词仅要求 LLM “分析失败原因并重写 Prompt，使其更像人类”。
2. **现象**：
   - **幻觉叠加 (Hallucination Stacking)**：DeepSeek 在没有被严格限制的情况下，会过度依赖其关于“什么是人类写作”的**先验知识**（Priors），导致其给出的建议越来越偏离数据反馈，反而引入了“假装口吃”、“加入口语化词汇 hmm/wait”、“完全不换行”等极端非自然口语化要求。
   - **生成器崩溃**：Kimi 面对这些极端的角色扮演提示词（Role-play prompts），指令遵循度极速下降，或者生成的排版陷入彻底的混乱，导致在第 3 轮绕过率达到 33.33% 后迅速回落至 0%，触发了早停机制。
   
### 阶段二：数据驱动型 Optimizer (Data-Driven Optimizer)
1. **策略**：针对上述发现，我们彻底重写了 Optimizer 和 Feedback 提示词。
   - **严格限制**：在 `optimizer_system_prompt` 中明确加入了 `CRITICAL RULE`，**禁止使用任何口语化幻觉或先验假设**。
   - **强显式映射**：在 `feedback_prompt` 中，把具体数值的偏差直接映射到了行动指南上（例如：“段落数 Generated 8.68 vs Human Target ~1.5 (Action: Constrain paragraph count)”）。
2. **现象与结果**：
   - **奇迹般的提升**：在第二轮，Optimizer 给出了极为结构化和严谨的指令（如明确列出“格式要求”、“数学符号上限”以及“禁用词黑名单”），而不是废话连篇的扮演设定。
   - **100% 绕过**：仅在第 2 轮，Kimi 在新的严格量化提示词的引导下，**瞬间实现了 100% 的检测绕过率 (15/15 evaded detection)**！

**核心洞察与结论**：
1. **“大模型认为的人类” ≠ “真实的人类数据分布”**：当让 LLM 自由发挥去“模仿人类”时，它往往会陷入刻板印象（如认为人类一定会写错别字或用很多语气词）。而高精度的分类器是基于真实人类的排版和行文特征（比如 1.5个平均段落、较低的逻辑连词密度）训练的。
2. **LLM 优化器必须被强行降维打击**：在构建 LLM Agent 迭代链路时，不能让优化器做过度发散的推理，而必须用**强显式的数据和规则**把它“按在地上摩擦”，逼迫它只做数值约束转化，这样才能得到真正有效的 Prompt。
3. **高精度分类器的终极软肋**：即便是高达 95.5% 准确率的、基于复杂专家经验特征构建的树集成模型，一旦其核心判断锚点被暴露给一个严格执行指令的 LLM，防线瞬间就会土崩瓦解。这说明，纯文本层面的 AI 痕迹检测在未来可能是一个注定要失败的防守战。
