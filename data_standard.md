# 数学题目数据集规范

## 1. 目标
为了方便大语言模型（LLM）API 读取、处理并生成答案，我们将原始的 LaTeX 题目文件统一转换为规范的 CSV 格式。该数据集主要用于区分不同来源（人类 vs 不同LLM）在数学解答上的特征差异。

## 2. CSV 字段定义
每条题目记录将被保存为一行，包含以下字段：

| 字段名 | 类型 | 说明 | 示例 |
|---|---|---|---|
| `id` | String | 题目的唯一标识符，格式为 `课程缩写_题号` | `ode_1` 或 `algebra_1` |
| `course` | String | 课程名称 | `常微分方程` 或 `近世代数` |
| `content` | String | 题目的 LaTeX 原文内容，包含多行时需保留换行符 | `判别下列方程是否为恰当方程...` |

## 3. 处理规则
1. **清理多余格式**：去除顶部的 `\documentclass`、宏包引入、`\begin{document}` 等环境标签。
2. **结构解析**：通过匹配 `\section`、`\subsection`、`\item` 以及特定题号格式（如 `1.1.1.`）来识别层级结构和每一道独立的题目。
3. **内容保留**：题干内的数学公式（如 `$...$`、`\[...\]`）和子列表（如 `\begin{enumerate} \item ... \end{enumerate}`）将被完整保留在 `content` 字段中。
4. **输出路径**：处理后的数据文件命名为 `processed_data.csv`，存放在根目录下的 `processed_data/` 文件夹内。
