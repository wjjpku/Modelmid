# 项目三：基于文本与逻辑特征的数学作业来源判别（Human vs 各类LLMs）

## 1. 项目背景
随着大语言模型（LLM）的普及，利用 ChatGPT、Kimi、文心一言等工具辅助完成数学理论推导作业或编程作业已成为常见现象。本项目的目标是通过批量采集人类真实作业与不同大语言模型生成的数学题解答，提取风格与逻辑特征，构建分类器，实现对解答来源的精准判别。

## 2. 数据处理说明
为了方便大语言模型 API 批量读取、处理并生成对应的数学题目解答，我们将收集到的原始 LaTeX 题目文件及人类已有解答进行了结构化清洗和规范转换。

### 2.1 原始数据
原始数据存放于 `data/` 目录下，包含：
- `ode题目latex代码`（《常微分方程》相关题目，仅题目）
- `近世代数群论部分题目及解答latex代码（前554行为题目）`（《近世代数》群论相关题目及解答）

### 2.2 处理脚本
我们编写了 `process_data.py` 脚本，该脚本能够：
1. 自动解析并过滤原始 LaTeX 文件中的格式控制标签。
2. 智能识别题目及其解答部分。例如，从近世代数文件中不仅提取了题目正文，还关联提取了其后续的人类答案。
3. 将提取的题目和解答组装成统一规范的数据格式，如果没有答案则保留为空。
4. 标注 `source`，如近世代数中的原始解答标注为 `Human`，其他空白数据可以后续由 `Deepseek` 等补全。

**运行方式**：
```bash
python3 process_data.py
```

### 2.3 输出规范
处理后的文件保存在 `dataset/full_dataset.csv`，共提取了 277 道题目及其现有的解答。详细的字段定义与规范请参考 [data_standard.md](data_standard.md)。
