# 数学题目数据集规范

## 1. 目标
为了方便大语言模型（LLM）API 读取、处理并生成答案，我们将原始的 LaTeX 题目及已有解答文件统一转换为规范的 CSV 格式。该数据集主要用于区分不同来源（人类 vs 不同LLM）在数学解答上的特征差异。

## 2. CSV 字段定义
每条记录包含一道题目以及对应的不同来源的解答。目前包含以下字段：

| 字段名 | 类型 | 说明 | 示例 |
|---|---|---|---|
| `id` | String | 题目的唯一标识符，格式为 `课程缩写_题号` | `ode_1` 或 `algebra_1.1.1` |
| `course` | String | 课程名称 | `常微分方程` 或 `近世代数` |
| `content` | String | 题目的 LaTeX 原文内容，包含多行时需保留换行符 | `判别下列方程是否为恰当方程...` |
| `human` | String | 题目对应的人类真实 LaTeX 解答正文。若暂无解答则为空 | `\textbf{证} (1) 由定义...` |
| `deepseek` | String | 题目对应的 Deepseek 模型生成的 LaTeX 解答正文。若暂无解答则为空 | `...` |
| `kimi` | String | 题目对应的 Kimi (Moonshot) 模型生成的 LaTeX 解答正文。若暂无解答则为空 | `...` |

## 3. 处理规则
1. **内容清理与解析**：去除顶部的格式控制标签，解析特定题号来提取每一道独立的题目。
2. **解答关联**：当原始文件中包含人类标准答案（如近世代数文件）时，会自动将解答正文提取到 `human` 字段。若暂无数据则留空。
3. **内容保留**：题目和解答内的数学公式及换行符均将被完整保留。
4. **输出路径**：处理后的数据文件命名为 `full_dataset.csv`，统一存放在项目根目录下的 `dataset/` 文件夹内。
