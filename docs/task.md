# 项目三：基于文本与逻辑特征的数学作业来源判别（Human vs 各类LLMs）

## 背景

随着大语言模型（LLM）的普及，利用ChatGPT、Kimi、文心一言等工具辅助完成数学学院的理论推导作业或编程作业（如科学计算课程）已成为常见现象。但不同的“大脑”（如不同的LLM版本或人类）在处理同一道复杂的数学证明题或编写代码时，就如同不同版本的“编译器”，其生成的“输出结果”在行文风格、逻辑跳跃程度、特定词汇使用频率、甚至潜在的漏洞特征上都会存在差异。

## 要求

通过批量采集人类学生撰写的以及不同大语言模型生成的数学题解答（或代码），提取风格与逻辑特征，
构建分类器，实现对解答来源的精准判别。实际研究中，可以逐次考虑如下问题：

1. **数据采集与特征提取** ：可以选取一门专业课程（如《数学分析》、《高等代数》或《概率统计》）的20
   道经典证明题。分别使用人类真实作业，以及至少3种不同大语言模型（如GPT-5, Kimi, Deepseek等等）
   生成答案。对比这些结果，找出区分它们（尤其是区分某特定模型与人类）的主要特征（如语句长度、
   连接词频率、公式排版特征、逻辑跳跃节点等）。
2. **构建判别函数/模型** ：结合问题一得到的特征，利用统计学方法或机器学习分类算法（如逻辑回归、SVM或随机森林），构造一个分类/判别模型，使得能够较大概率准确判断某份作业是由哪个大模型生成，或是人类独立完成的。
3. **泛化性研究** ：用问题二得到的模型，去判别另一门完全不同风格课程（如《数据结构与算法》或《常微分方程》）的解答来源。研究该判别模型在跨学科、跨题型时的有效性。

**建议：** 可以延伸研究防检测与反制干预，基于你们对判别器的研究，给出几条让 LLM 生成的数学作业更像“人类学生手写”的建议（即如何对抗这种判别器）。
